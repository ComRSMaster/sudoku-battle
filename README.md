# Sudoku Battle

## 1. Идея проекта
**Sudoku Battle** - это соревновательная игра в Telegram на базе **Mini Apps**.

В игре есть 4 режима:

1. **Одиночный режим**, где игрок в одиночку разгадывает случайное поле.
2. **Судоку дня**, где игрок разгадывает на скорость поле, которое каждый день одинаковое для всех игроков.
3. **Режим «Дуэль»**, где два игрока сражаются, решая одно и то же поле судоку на скорость.
4. **Командный режим**, где несколько игроков вместе решают одно и то же поле.

Игра включает в себя бэкенд на Python для обработки сообщений в Telegram, генерации валидных пазлов, обработки ходов в реальном времени и веб-интерфейс для удобного взаимодействия с полем.

## 2. План разработки по итерациям

### Итерация 1
* **Фича 1: Basic game logic.** Разработка ядра игры на Python: алгоритм генерации корректных судоку. Реализация логики проверки ходов и состояния победы.
* **Фича 2: Backend API (FastAPI).** Разработка асинхронного сервера для взаимодействия с Web App. Эндпоинты для получения нового поля, валидации хода и хранения промежуточного состояния игры.
* **Фича 3: Базовый Telegram Bot.** Интерфейс запуска игры через команду `/start` и кнопку `WebApp`.

### Итерация 2
* **Фича 1: Real-time Multiplayer.** Реализация мультиплеера. Использование WebSockets для синхронизации прогресса игроков во время игры.
* **Фича 2: Web App Interface.** Полноценный фронтенд на Svelte: интерактивная сетка 9x9, таймер.
* **Фича 3: Лидерборд.** Общий рейтинг всех игроков в боте (по количество разгаданных судоку) и ежедневный рейтинг пользователей в режиме «Судоку дня».

## 3. Используемые библиотеки для бэкенда

`aiogram` (для бота), `FastAPI` (для API), `websockets` (для мультиплеера).

### Как запустить

Скопировать файл `.env.example` в `.env`.

Поменять в нем `BOT_TOKEN` на свой токен Telegram-бота, этот токен можно взять в BotFather.

Поменять в нем `WEB_APP_URL` на свой URL-адрес, на котором планируется разворачивать игру.
Если нет своего URL-адреса, можно воспользоваться Ngrok или Port forwarding в VS Code.

Команда для запуска: `uv run main.py`.

`uv` сам загрузит все необходимые зависимости.
